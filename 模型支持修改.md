图片处理：

直接截取屏幕 (Screenshot)
这是目前很多主流 AI 插件（如 Monica, Sidebar）的做法。不解析单个图片，而是直接把用户当前看到的网页整个截图发给模型。
// background.js
chrome.tabs.captureVisibleTab(null, { format: 'jpeg', quality: 50 }, (dataUrl) => {
  // dataUrl 就是当前窗口的 Base64 截图
  // 直接把它发给 gpt-4o 或 claude-3-5-sonnet
  callVisionModel(dataUrl, pageText);
});

优点： 完美保留排版，AI 能看到图表、样式和图片。
实现方法： 使用 Chrome 浏览器 API：

如果你要实现一个功能完备的插件，建议采取 “文本提取 + 屏幕截图” 的组合拳：

文本提取：通过 innerText 获取核心语义，节省 Token。

屏幕截图：调用 chrome.tabs.captureVisibleTab 获取视觉信息。

配置化：在设置页面增加一个开关 “启用图片分析”。

因为图片 Base64 非常占 Token（一张图可能消耗几百到上千 Token），且多模态模型（Vision）比普通模型贵，一定要给用户选择权。
