开发一个AI插件这样能“读取页面内容”并支持自定义模型协议（OpenAI/Anthropic）的浏览器插件，你需要掌握 Chrome Extension Manifest V3 的核心机制。

以下是一个简化版的实现方案，包含了获取页面内容、配置模型以及发送请求的核心代码。
创建一个文件夹，包含以下文件：

manifest.json：插件配置文件

popup.html：用户设置界面（配置 API Key 和协议）

popup.js：处理设置保存

content.js：关键文件，运行在网页环境，负责提取内容

background.js：负责与 AI 服务器通信（避开跨域问题）

2. 核心代码实现
第一步：配置文件 (manifest.json)
你需要申请 activeTab 和 scripting 权限来读取网页。

{
  "manifest_version": 3,
  "name": "AI Page Chat",
  "version": "1.0",
  "permissions": ["activeTab", "scripting", "storage"],
  "action": {
    "default_popup": "popup.html"
  },
  "background": {
    "service_worker": "background.js"
  }
}
第二步：获取页面内容 (content.js)
这个脚本会被注入到当前页面，抓取所有的文本。
// content.js
function getPageContent() {
  // 简单暴力获取所有文本，进阶版可以使用 Readability.js 提取正文
  return document.body.innerText || "";
}

// 监听来自插件弹窗的消息
chrome.runtime.onMessage.addListener((request, sender, sendResponse) => {
  if (request.action === "getText") {
    sendResponse({ content: getPageContent() });
  }
});

第三步：后台通信 (background.js)
在这里处理不同协议（OpenAI/Anthropic）的逻辑。
// background.js
chrome.runtime.onMessage.addListener((request, sender, sendResponse) => {
  if (request.action === "askAI") {
    const { protocol, apiKey, model, prompt, context } = request.data;
    
    callAI(protocol, apiKey, model, prompt, context).then(sendResponse);
    return true; // 保持异步连接
  }
});

async function callAI(protocol, apiKey, model, prompt, context) {
  const url = protocol === "openai" 
    ? "https://api.openai.com/v1/chat/completions"
    : "https://api.anthropic.com/v1/messages";

  const headers = { "Content-Type": "application/json" };
  let body = {};

  if (protocol === "openai") {
    headers["Authorization"] = `Bearer ${apiKey}`;
    body = {
      model: model,
      messages: [{ role: "user", content: `Context: ${context}\n\nQuestion: ${prompt}` }]
    };
  } else {
    headers["x-api-key"] = apiKey;
    headers["anthropic-version"] = "2023-06-01";
    body = {
      model: model,
      max_tokens: 1024,
      messages: [{ role: "user", content: `Context: ${context}\n\nQuestion: ${prompt}` }]
    };
  }

  const res = await fetch(url, { method: "POST", headers, body: JSON.stringify(body) });
  return await res.json();
}

第四步：前端交互 (popup.js)
连接一切：获取网页内容 -> 读取配置 -> 发送给 background。

// popup.js 里的核心逻辑
async function handleQuery() {
  // 1. 从当前标签页获取文本
  const [tab] = await chrome.tabs.query({ active: true, currentWindow: true });
  const response = await chrome.tabs.sendMessage(tab.id, { action: "getText" });
  const pageText = response.content;

  // 2. 获取用户配置的 API 信息 (假设已通过 storage 存储)
  const config = await chrome.storage.local.get(["apiKey", "protocol", "model"]);

  // 3. 发送给后台处理请求
  chrome.runtime.sendMessage({
    action: "askAI",
    data: {
      ...config,
      context: pageText,
      prompt: "请总结这个页面的核心内容"
    }
  }, (aiRes) => {
    console.log("AI 回复:", aiRes);
    document.getElementById("output").innerText = JSON.stringify(aiRes);
  });
}

3. 实现场景的几个难点（建议）
内容清洗：直接拿 innerText 会包含导航栏、广告等噪音。建议引入 Mozilla 的 Readability.js 库，它能像“阅读模式”一样只提取文章正文。

Token 溢出：长网页内容会轻松超过 LLM 的上下文限制。你需要简单的文本截断逻辑或者前端 RAG（向量化检索）。

流式传输 (Streaming)：为了像我一样“一个字一个字”蹦出来，你需要使用 fetch 的 ReadableStream 并在 background 和 popup 之间建立 chrome.runtime.connect 长连接。

跨域问题：Manifest V3 中，background.js 使用 fetch 不受同源策略限制，但必须在 host_permissions 中声明 API 地址。
